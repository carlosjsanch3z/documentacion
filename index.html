Instalación

Para empezar con un proyecto usando Jeykill en GitHub Page, hay varias opciones, unas más complejas y limpias y otras más rápidas de forma que nuestro proyecto estará disponible y visible antes.

Opción 1

Instalar Jeykill de manera local mediante linea de comandos, haciendo uso de jekyll new, construyendolo localmente con jekyll build. En el siguiente enlace te lo explica con más detalles.

Opción 2

Clonar un punto de inicio en tú máquina local, instalar Jekyll localmente mediante linea de comando, hacer los cambios en tú web, contruirlo y publicarlo.

Opción 3

Bifurcar un punto de partida, realizar los cambios y publicarlo. Esta es la opción más rápida y fácil de poner en marcha la página web estática en GitHub Page. Sin necesidad de instalar ninguna dependencia. Incluso pudiendo hacerlo todo desde el navegador en GitHub.com

  1. Crear nuestro sitio con una plantilla predeterminada de GitHub.
  2. Publicarlo gratis en GitHub Pages.
  3. Personalizarlo esteticamente modificando el fichero .css e incluyendo contenido con el lenguaje Markdown.

Demostración "Usando una plantilla de GitHub Pages"

Nos registramos en GitHub e iniciamos sesión. Una vez hayamos iniciado sesión, creamos un repositorio nuevo https://github.com/new

1. Creando el repositorio correctamente

Si queremos que nuestra página web estática aparezca directamente en la url: https://carlosjsanch3z.github.io, el nombre del repositorio deberá ser exactamente el nombre de nuestro usuario en github. En mi caso carlosjsanch3z.github.io


En el ejemplo de la foto aparece username.github.io debido a que la documentación la he redactado después de realizar esta práctica y por lo tanto es imposible crear un nuevo repositorio con el mismo nombre de uno ya creado.

2. Eligiendo la rama maestra

A continuación de crear el nuevo repositorio, nos situamos en él y nos desplazamos al botón "Settings" que se encuentra en la parte superior del menú del repositorio. Una vez ahí desplazamos la rueda del ratón hacía abajo, hasta la ventana nombrada como GitHub Pages.


En la casilla, elegimos la opción Master branch para que use la estructura de ficheros que se encuentran en la raíz del repositorio y así poder ser publicada posteriormente

3. Generando la página de forma automática

Simplemente clickearemos en el botón "Launch automatic page generator" Y nos aparecerá para indicar algunos parametros como por ejemplo el nombre del proyecto y nos mostrará algunos ejemplos de la sintaxís Markdown.


4. Seleccionando plantilla predeterminada

En la siguiente ventana, visualizaremos una previsualización de nuestro proyecto dependiendo de la plantilla que seleccionemos.


Tendremos la posibilidad de elegir entre unas 10 plantillas diferentes que aplicarle a nuestro nuevo proyecto. Cuando nos decantemos por una de ellas, clikearemos en el botón "Publish page".

5. Estructura de ficheros final y página web publicada

En la siguiente imagen vemos como es la estructura final de los ficheros que nos crea automáticamente este generador, por parte de GitHub.


Finalmente podemos visualizar esta página web estática creada de ejemplo para esta documentación desde el siguiente enlace https://carlosjsanch3z.github.io/username.github.io/. Si nos fijamos detenidamente la url no es del todo amistosa, esto se debe a como bien explicamos en el punto 1 de este apartado, el nombre del repositorio debe ser el mismo que nuestro usuario en Github seguido de .github.io de esa forma conseguiremos que la URL sea limpia como en mi práctica real, https://carlosjsanch3z.github.io/
